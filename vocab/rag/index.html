RAG

Purpose

RAG (red-amber-green) is a simple colour-coded system for indicating status or risk levels.

Implementation notes

Integer or floating-point numbers?

The transitions between values are often the most important information for end users. Floating point values allow changes to be tracked over time rather than rounding off in a lossy manner. A project that was tracking at 1.0 (green) last week and is now 1.3 is still green, but it may be useful to know about the change before it breaches 1.5 (amber).

Values SHOULD be checked against the range using minimum (inclusive) and maximum (exclusive) bounds:

// [min, max) — min inclusive, max exclusive
value >= min && value < max

  • The canonical integer values (1, 2, 3) sit at the centre of each concept's range and are sufficient for most use cases. Implementations MAY choose to use integers only.
  • Values cross into the next concept at the boundary. A value of 1.5 is amber, not green.
  • Considering floating point values:
    • allows users to organise items via drag and drop, placing items relative to each other based on their knowledge.
    • can calculate roll up status based on multiple child items for greater granularity in reporting.
  • 0 up to 0.5 (exclusive) is not defined. This is intentional to allow 1 to sit consistently in the middle of the range.
  • Values below 0.5 or above 3.5 fall outside the defined range of this vocabulary.
  • Colours are illustrative only. Implementors SHOULD choose colours (if any) based on usability and accessibility.
